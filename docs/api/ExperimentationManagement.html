• Overview

OrganiCity Experimentation Management

Version: 1.0

This document specifies the API to programmatically manage experiments running on the OrganiCity platform. Furthermore the API provides suitable methods for other service to get experiments-related information.

The API comprises the following functionalities:

  • Experiments Creation
  • Applications Creation
  • Assets selection
  • Assets creation
  • Facility-level
  • Experiments Checks

Schemes: http

Base URL:

Summary

Experiments

Path Operation Description
/experiments GET

Get the list of experiments in which one experimenter is involved, either as main experimenter or as invited one. The experimenter ID is obtained from the bearer token
/experiments POST

Create a new experiment. The experimenter ID is obtained from the bearer token and it will be the ID of the main experimenter
/experiments/{expId} GET

Retrieve a single experiment indicated by expId. The experimenter ID is obtained from the bearer token. Note that any experimenter associated to the experiment can get this information
/experiments/{expId} PUT

Update the information of the experiment indicated by expId. The experimenter ID is obtained from the bearer token.
/experiments/{expId} DELETE

Delete the experimenter indicated by expId. The experimenter ID is obtained from the bearer token. Note that only the main experimenter is allowed to delete it
/experiments/addexperimenter/{expId}/{email} POST

Add an experimenter to the development group of the experiment indicated by expId. The new experimenter is defined by the email used to register in Organicity as experimenter. Any experimenter of the experiment is allowed to add more experimenters
/experiments/removeexperimenter/{expId}/{experId} POST

Remove experimenter experId from the development group of the experiment indicated by expId.

Applications

Path Operation Description
/experiments/{expId}/applications GET

Retrieve the list of the applications belonging to the experiment indicated by expId. Only experimenters within the development team of the experiment are allowed to obtain this information (the experimenter ID is obtainbed from the bearer token)
/experiments/{expId}/applications POST

Create a new application within the experiment indicated by expId. Note that only experimenters belonging to the edevelopment group of the experiment are allowed to do it (the experimenter ID is obtainbed from the bearer token)
/experiments/{expId}/applications/{appId} GET

Get the application with identifier appId belonging to the experiment indicated by expId. Note that only experimenters within the development group of the experiment are allowed obtain this information (the experimenter ID is obtainbed from the bearer token)
/experiments/{expId}/applications/{appId} PUT

Update the application with identifier appId that belongs to the experiment indicated by expId. Note that only experimenters within the development group of the experiment are allowed obtain this information (the experimenter ID is obtainbed from the bearer token)
/experiments/{expId}/applications/{appId} DELETE

Delete the application with identifier appId that belongs to the experiment indicated by expId. Note that only experimenters within the development group of the experiment are allowed obtain this information (the experimenter ID is obtainbed from the bearer token)

Assets Selection

Path Operation Description
/experiments/{expId}/datasources GET

Get the assets selected to be used within the experiment (datasources) indicated by expId. This assets do not include those created by the experiment, but public assets chosen to enrich the experiment inforamtion. Note that only experimenters within the development group of the experiment are allowed obtain this information (the experimenter ID is obtainbed from the bearer token)
/experiments/{expId}/datasources PUT

Get the assets selected to be used within the experiment (datasources) indicated by expId. Note that only experimenters within the development group of the experiment are allowed obtain this information (the experimenter ID is obtainbed from the bearer token)
/experiments/{expId}/devices GET

Get the list of assets created within the experiment indicated by expId. These assets can have been created by any application belonging to the experiment, from the Experimenter Portal or by using offline token. Note that only experimenters within the development group of the experiment are allowed obtain this information (the experimenter ID is obtainbed from the bearer token)
/experiments/{expId}/devices POST

Create a new asset within the experiment indicated by expId. This method adds the attribute TimeInstant (compulsory for all assets) or overwrites the current content of that attribute setting the current time. Note that only experimenters within the development group of the experiment are allowed obtain this information (the experimenter ID is obtainbed from the bearer token)

Facility-level management

Path Operation Description
/experiments-full/{expId} GET

Get all the information ceoncerning the experiment indicated by expId. This information includes the list of applications, data-sources (selected assets) and devices (created assets). This method is intented to be used ONLY by the Facility Management Service, thus only queries from such service are allowed.

/allexperiments GET

Get a list of all the experiments created within the Organicity facility. This method is intented to be used ONLY by the Facility Management Service, thus only queries from such service are allowed.

/experiments/{expId}/remainingquota GET

Get the remaining quota of the experimenter indicated by expId. The remaining quota indicated the number of assets that can be still created by the experiment (e.g. by its applications). Note that only experimenters within the development group of the experiment (along with some Organicity services) are allowed obtain this information (the experimenter ID is obtainbed from the bearer token)

/experiments/{expId}/increaseremquota POST

Increaset by one unit the remaining quota of the experiment indicated by expdId. This resource is intended to be ONLY by Organicity facility services, thus queris from external services will be ignored.

/experiments/{expId}/decreaseremquota POST

Decrease by one unit the remaining quota of the experiment indicated by expId. This resource is intended to be ONLY by Organicity facility services, thus queris from external services will be ignored.

Experiments check

Path Operation Description
/emscheck/experimenter-experiment/:experId/:expId GET

Check wether the experimenter with identifier experId belongs to the development group of the experiment indicated by expId. This resource is intended to be ONLY by Organicity facility services, thus queris from external services will be ignored.

/emscheck/experimentrunning/:expId GET

Check wether the experiment indicated by expId is running. This resource is intended to be ONLY by Organicity facility services, thus queris from external services will be ignored.

/emscheck/application-experiment/:expId/:appId GET

Check wether the application with indentifier appId belongs to the experiment indicated by expId. This resource is intended to be ONLY by Organicity facility services, thus queris from external services will be ignored.

/emscheck/assets-public/:expId GET

Check wehther or not the assets created within the experimenter indicated by expId are public. This resource is intended to be ONLY by Organicity facility services, thus queris from external services will be ignored.

/emscheck/participant-experiment/:parId/:expId GET

Check whether the Organicity user with identifier parId is registered as a participant of the experiment indicated by expId. This resource is intended to be ONLY by Organicity facility services, thus queris from external services will be ignored.
